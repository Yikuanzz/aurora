# Task Card Template — Aurora 任务卡片模板

> 新建任务时复制此模板，保持格式一致。

```markdown
## 任务标题

一句话描述要做什么。

## 背景/动机

为什么需要这个任务？它解决什么问题？

## 验收标准

- [ ] 标准 1：具体可验证的行为
- [ ] 标准 2：具体可验证的行为
- [ ] 标准 3：边界情况处理

## 涉及文件

| 层级 | 文件 | 改动类型 |
|------|------|----------|
| 前端 | `src/...` | 新增/修改 |
| 后端 | `src-tauri/src/...` | 新增/修改 |
| 类型 | `src/types/...` | 新增/修改 |

## 依赖任务

- #xxx (如果有前置任务)

## 估计工作量

小 / 中 / 大

## 备注

任何额外需要注意的事项。
```

---

## 示例

```markdown
## 实现记一笔表单 (LogForm)

完成用户记录每日行动的弹窗表单。

## 背景/动机

MVP 核心功能。用户需要通过悬浮按钮快速记录目标进度。

## 验收标准

- [ ] 表单字段：目标选择器、日期选择器、类型切换(count/time/boolean)、标签输入、数值输入、单位输入、感受文本框
- [ ] 提交后调用 `db_add_log` 命令写入数据库
- [ ] 成功后有视觉反馈（进度条充能动画）
- [ ] 表单验证：目标必选，数值必填
- [ ] 支持编辑已有记录

## 涉及文件

| 层级 | 文件 | 改动类型 |
|------|------|----------|
| 前端 | `src/components/LogForm.tsx` | 大幅修改 |
| 类型 | `src/types/index.ts` | 可能无需改动 |

## 依赖任务

- 无

## 估计工作量

中

## 备注

注意与 GoalStore / LogStore 的联动更新。
```
