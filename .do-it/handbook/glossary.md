# Glossary — Aurora 术语表

> 项目中特定词汇的统一解释，避免沟通歧义。

## 角色与概念

| 术语 | 解释 |
|------|------|
| **指挥官** | 对用户的称呼，强化"指挥模拟游戏"的沉浸感 |
| **Aurora** | AI 伙伴的名字，战友型女友定位 |
| **指挥舱** | 应用主界面/首页，用户进入后的第一视图 |
| **星图** | 目标可视化沙盘，用抽象星系表示目标关系 |
| **星系** | 一个目标及其关联里程碑的集合，星图上的一个节点簇 |

## 功能模块

| 术语 | 解释 |
|------|------|
| **目标 (Goal)** | 用户要达成的大目标，星图中的主要节点 |
| **里程碑 (Milestone)** | 目标的阶段性检查点，默认顺序解锁 |
| **行动 (Action)** | 里程碑下的具体可执行建议，规划层 |
| **记一笔 (Log)** | 用户记录每日行动的入口，对应 `daily_logs` 表 |
| **同步率** | UI 中展示的概念指标，暗示用户与 Aurora 的配合度 |
| **精力值** | UI 中展示的概念指标，暗示用户的投入状态 |

## 数据类型

| 术语 | 解释 |
|------|------|
| **count** | 日志类型：计数型（如"做了 5 道题"） |
| **time** | 日志类型：时间型（如"学习了 30 分钟"） |
| **boolean** | 日志类型：布尔型（如"今天运动了"） |
| **embedding** | 向量表示，用于 AI 记忆的语义检索 |
| **importance_score** | AI 记忆的重要性评分，影响遗忘和检索优先级 |

## 技术术语

| 术语 | 解释 |
|------|------|
| **Tauri** | 用 Rust 构建跨平台桌面应用的框架 |
| **Rusqlite** | Rust 的 SQLite 绑定库 |
| **Zustand** | React 的轻量级状态管理库 |
| **Stronghold** | Tauri 的加密存储插件，用于保护 API Key |
| **Tavily** | 搜索引擎 API，Aurora 分享内容的数据源 |
| **流式输出** | AI 响应逐字返回，而非等待完整响应 |

## 情感/状态术语

| 术语 | 解释 |
|------|------|
| **微步引导** | Aurora 的核心策略：用户想放弃时，只鼓励做 1 分钟 |
| **克制的愤怒** | 用户熬夜时 Aurora 的情绪表现 |
| **情绪奖赏** | 用户完成小步骤后 Aurora 给予的正向反馈 |
| **隐性好感度** | 用户不可见的好感度数值，只通过语气变化体现 |
